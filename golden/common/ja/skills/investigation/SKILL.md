# 原因調査スキル

## 発火条件
原因調査・バグ調査・問題の切り分けを行う際に発火する。

## 基本原則
- 調査フェーズと実装フェーズは分離する
- 推測で回答するな。実行時の事実で裏取りしてから報告
- 「可能性がある」「はず」で終わる調査結果は不合格

## コードパス全数調査
1. 関連する全コードパスの列挙（1件見つけたは調査の開始であり終了ではない）
2. 実行パスの特定（デバッグログ・import chain・実行時証拠で確認）
3. 全パスの状態比較（差異を表形式で整理）

## 反証チェック（結論を報告する前に必須）
自分の結論を否定する証拠を能動的に探すこと。

```
## 反証チェック
- [ ] 同じ機能を持つコードパスは他にないか？
- [ ] 実行時ログで結論を裏付けたか？
- [ ] この結論が間違っていた場合、何が真の原因か？（代替仮説 1 つ以上）
- [ ] 過去の類似調査で初回結論が間違っていた事例はないか？
```

## 調査結果まとめ（必須出力）
```
=== 調査結果まとめ ===
- 原因：
- 影響範囲：
- 修正方針：
```
