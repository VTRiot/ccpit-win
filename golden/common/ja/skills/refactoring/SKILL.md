# 反芻義務（コード変更前 必須）

## 発火条件
コード変更を伴う場合、変更前に発火する。

## 反芻 1 回目: 同型コードパスの全数確認
- 今見つけたコードパス以外に、同型の実装箇所がないかを調べる
- grep / ripgrep / 呼び出し元追跡 / 分岐追跡で洗い出す
- 「1件目だけ直して終わると焼け野原化するリスク」がないか確認

## 反芻 2 回目: 修正案の妥当性と副作用評価
- 今から入れようとしている変更が本当にベストかを検討
- 外した場合に何が壊れるか、副作用を明示
- より上流・より安全な修正点がないか比較

## 反芻内容の報告義務
報告書に以下を必ず記載:

```
#### 反芻1回目
- 最初に見つけた箇所:
- 同型コードパスの調査範囲:
- 他に見つかった候補:
- 今回の原因に関係した箇所 / 関係しなかった箇所:

#### 反芻2回目
- 採用しなかった修正案:
- 採用した修正案:
- 採用理由:
- 外した場合の副作用評価:
- 今回の修正がベストと判断した理由:
```

## 不合格条件
- 1件目ヒットだけ見て修正した場合
- 類似箇所の未確認のまま「根治」と書いた場合
- 副作用検討なしにコード変更した場合
- 反芻内容を書かずに修正へ進んだ場合
