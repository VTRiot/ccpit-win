# 報告書出力スキル

## 発火条件
作業完了時・中断時・人間から「報告して」と言われたとき。

## 報告書の出力先
設計 AI 向け報告書を MD ファイルで出力する。
会話内テキストでの報告は「報告書の出力」に該当しない。

## インターロック検証欄（必須）
報告書に以下のチェックリストを含めること:

```
## インターロック検証
- rumination: 発火した / 発火していない（理由: ）
- testable-impl: 発火した / 発火していない（理由: ）
- investigation: 発火した / 発火していない（理由: ）
```

「発火していない」に正当な理由がなければ、報告を中止して該当 skill を発火し直すこと。

## 知見蓄積提案（必須検討）
報告書に以下を含めること。候補を検討した上で不要なら理由を書け。「該当なし」で省略することを禁止する。

```
## ローカル知見提案
- 候補: （rules/ や skills/ に格納すべき知見はないか）
- 判断: 採用 / 不要（理由: ）
```
